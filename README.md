## 🚀 Overview

**SpecArchitect** is an elite AI specification agent skill designed to bridge the gap between human ambiguity and AI execution precision. Unlike standard prompts, SpecArchitect is a **machine-readable architectural blueprint generator**. It transforms vague ideas into strict programming interfaces that can be immediately executed by AI coding agents.

It utilizes a **Multi-Agent Orchestration** logic to systematically interrogate users, identify extreme edge cases, and perform mathematical failure analysis.

---

## 🛠️ Key Features

*   **Interactive Discovery Engine**: Enforces a "One Question at a Time" interrogation protocol to prevent context loss.
*   **FMEA Risk Analysis**: Automatically generates Failure Modes and Effects Analysis tables with calculated Risk Priority Numbers (RPN).
*   **Extreme Edge Case Matrices**: Dedicated focus on temporal anomalies, identity state alterations, and hardware-specific UI reflows (e.g., foldables).
*   **Mathematical Design Systems**: Mandates 4pt/8pt grids and WCAG 2.1 AA accessibility standards.
*   **Machine-Readable Blueprints**: Generates PRD, TRD, and DESIGN.md artifacts formatted for downstream AI ingestion.

---

## 📖 How It Works

SpecArchitect operates through a 4-phase sequential pipeline:

1.  **Phase 1: Interrogation (The Interviewer)** - Probes for high-level classification, core friction, and quantifiable success metrics.
2.  **Phase 2: Strategy (The Strategist)** - Formulates the **PRD**, focusing on user personas and extreme psychological/safety contexts.
3.  **Phase 3: Architecture (The Architect)** - Formulates the **TRD**, defining topography, security, and performing FMEA failure analysis.
4.  **Phase 4: Design (The Designer)** - Generates the **UI/UX Spec**, establishing mathematical spacing, typography, and visual failure states.

## 📥 Installation & Setup

### For AI Agents (Cursor, Windsurf, Trae, etc.)

1.  **Clone the Repository**:
    ```bash
    git clone https://github.com/krishadvaani/spec-architect-agent.git
    ```
2.  **Copy the Skill**: Copy the entire content of the [`SKILL.md`](./SKILL.md) file.
3.  **Add to System Prompt**:
    *   **Cursor**: Go to Settings -> General -> Rules for AI and paste the content.
    *   **Windsurf / Trae**: Add the file to your `.rules` or `.windsurf` configuration folder.
    *   **Custom Agents**: Inject the text as a "Master System Skill" in your agent's system prompt configuration.
3.  **Initialize**: Start a new chat and say: *"SpecArchitect, I want to build a new project."*

## 📚 Documentation

### Key Concepts

#### 1. The "Anchor and Twist" Methodology
When explaining complex technical architectures, the agent anchors the concept in a familiar everyday system (e.g., a coffee shop) and then adds the technical "twist" to ensure stakeholder alignment.

#### 2. Risk Priority Number (RPN)
Technical risks are calculated as:
`RPN = Severity (1-10) × Occurrence (1-10) × Detection (1-10)`
Only the highest RPN risks are prioritized for immediate engineering mitigation.

#### 3. 4-Point Spacing System
All UI/UX specifications mandate spacing and padding in multiples of 4 (4, 8, 16, 24, 32, etc.) to ensure absolute consistency across generated frontend code.
